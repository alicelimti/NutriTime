### 작업 내용

#### 1. 프로젝트 초기 세팅
- GitHub 저장소 클론 (`alicelimti/NutriTime`)
- Vite + React 프로젝트 스캐폴딩
- Firebase SDK (v12) 설치 및 `src/firebase/config.js` 설정 파일 구성
- `.env.local` 환경변수 템플릿 생성 (`.gitignore`에 자동 제외)
- GitHub Pages 배포를 위한 `vite.config.js` base 경로 설정 (`/NutriTime/`)
- `gh-pages` 패키지 연동 (`npm run deploy`)

#### 2. 기획안 분석 (NutriTime.pdf)
PDF 기획안 9페이지를 분석하여 다음 6개 섹션의 요구사항 파악:

1. **앱 개요 및 핵심 가치** — 정확한 정보·스마트 알림·카카오 연동·건강 관리 4대 가치
2. **영양제 라이브러리** — 이모지·이름·효능 카드 UI, 상세 정보 모달
3. **복용 시간대 관리** — 기상직후~취침직전 5단계 그룹
4. **개인화 알림 시스템** — 30분 간격 설정, 복용 체크, 스마트 재알림
5. **카카오톡 알림 연동** — 플러스친구 메시지 발송 (Phase 2)
6. **수익화 전략** — iHerb·쿠팡·올리브영·GNC 제휴 (Phase 3)

#### 3. 파일 구조 설계 및 구현

#### 4. 주요 기능 구현

**영양제 라이브러리**
- 15종 영양제 카드 그리드 (2열)
- 이름·효능 검색 필터
- `+` 버튼으로 기본 시간대에 즉시 추가 / 체크(✓) 표시로 제거
- 카드 탭 시 바텀시트 상세 모달 표시

**영양제 상세 모달**
- 상세 설명 / 좋은 조합 / 추천 연령대 / 복용 시간대 / 주의사항
- 하단 시간대 선택 버튼 — 탭하면 해당 그룹으로 이동

**내 일정 (복용 스케줄)**
- 5단계 시간대 그룹 (기상직후·오전·오후·저녁·취침직전)
- 오늘 복용 진행률 바 (체크수 / 전체)
- 각 영양제 체크 토글 (오늘 날짜 기준 초기화)
- 영양제 제거 버튼 (호버 시 노출)

**설정**
- 기상·아침·점심·저녁·취침 시간 `<input type="time">` 설정
- 푸시 알림 토글 UI
- 카카오톡 알림 연동 플레이스홀더 (Phase 2 예정)
- 제휴 파트너 섹션 (iHerb·쿠팡·올리브영·GNC)

**상태 관리**
- `localStorage` 기반 영구 저장 (영양제 일정 / 시간 설정 / 오늘 체크 내역)
- 오늘 체크 내역은 날짜 기준 자동 초기화

### 기술 이슈 및 해결

**초기 배포 시 백지 화면 문제**
- 원인: `height: 100%` 체인(html→body→#root→.app)이 일부 브라우저에서 0px로 계산됨
- 해결: `.app`에 `min-height: 100vh; min-height: 100dvh` 적용,  
  `.page-content`에 `min-height: 0` 추가
- JSDOM 헤드리스 렌더 테스트로 JS 정상 실행 확인 후 CSS 원인 특정

**GitHub Pages 브랜치 설정 오류**
- 원인: Pages가 `main` 브랜치 루트(`src="/src/main.jsx"` 포함된 개발용 index.html)를 서빙
- 해결: GitHub REST API (`PUT /repos/alicelimti/NutriTime/pages`)로 소스 브랜치를 `gh-pages`로 변경

#### 주요 기능

**체중 기반 권장 음수량 계산**
- 상단 카드에 체중(kg) 입력 → `체중 × 30ml` 공식으로 권장 음수량 즉시 계산
- 체중값은 `localStorage`에 영구 저장

**SVG 투명 물병 시각화**
- 순수 SVG로 구현한 유리 물병 모양 (네크 + 어깨 곡선 + 몸통 + 둥근 바닥)
- 글래스 효과: 반투명 테두리(`rgba(100,181,246,0.65)`), 내부 세로 하이라이트 shine
- 물 채워짐: 하단에서 위로 애니메이션 (`cubic-bezier(.34,1.56,.64,1)` 스프링 효과)
- 수면 파도 애니메이션: SVG `<animateTransform>` 으로 타원이 좌우로 흔들림
- 100ml 단위 눈금 틱 마크, 500ml 단위 레이블(1L 형식) 표시
- 물병 최상단에 목표 음수량 배지(`목표 Nml`) 항상 노출
- 물 채워진 비율이 18% 초과 시 물병 내부에 현재 음수량 텍스트 표시

**음수량 기록 조작** *(초기 구현 — 이후 Range Slider로 교체됨)*
- `+100ml` / `-100ml` 버튼으로 100ml 단위 기록
- 원형 진행률 SVG 그래프로 달성률(%) 중앙 표시
- 초기화 버튼 (소비량 > 0 일 때만 노출)
- 목표 달성 시 🎉 팝인 애니메이션 배너 표시

**상태 관리**
- 음수량은 오늘 날짜(`toDateString()`) 기준 자동 초기화
- 체중은 날짜 무관 유지

#### 상태 관리 구조 (최종)

```
localStorage
├── mySupplements   { wake:[], morning:[], afternoon:[], evening:[], bedtime:[] }
├── myMedications   { wake:[], morning:[], afternoon:[], evening:[], bedtime:[] }
├── settings        { wakeTime, breakfastTime, lunchTime, dinnerTime, bedtimeTime }
├── todayChecked    { date: "...", checked: ["vitd", "med_bp", ...] }
├── waterWeight     "65"  (체중, 문자열)
└── waterConsumed   { date: "...", amount: 1500 }
```

#### 디자인 결정

- **파란 계열 헤더**: 음수량 페이지 헤더를 파란 그라디언트(`#0D47A1 → #1976D2 → #42A5F5`)로 처리. 영양제(초록) · 복용약(하늘파랑)과 구분되는 짙은 파랑으로 물 이미지 강조.
- **SVG 선택 이유**: CSS만으로는 어깨 곡선이 있는 병 모양의 clipPath 구현이 어려움. SVG `<path>` + `<clipPath>`로 정확한 병 모양 안에 물 채움 효과 구현.
- **5탭 레이아웃**: 탭이 4개 → 5개로 늘어나 각 탭의 가로 폭이 줄어들지만, `font-size: 11px` 레이블과 `22px` 아이콘으로 480px 최대 너비 내에서 균형 유지.

### 작업 배경

버튼(+100ml / -100ml) 방식은 100ml 단위로만 입력 가능해 세밀한 기록이 불가.  
맥북 볼륨 조절처럼 드래그로 연속적으로 조절하는 아날로그식 UI로 개선 요청.

### 변경된 파일

| 파일 | 변경 내용 |
|------|-----------|
| `src/pages/WaterPage.jsx` | 버튼·원형 링 제거 → 세로 슬라이더 + `bottle-and-slider` 레이아웃으로 교체 |
| `src/pages/WaterPage.css` | 슬라이더 전용 스타일 추가, 버튼 관련 스타일 전면 제거 |

### 구현 방식

**세로 슬라이더 CSS 기법**
- `writing-mode: vertical-lr; direction: rtl` — 세로 방향 슬라이더, 위=최대값·아래=0
- `step="10"` / `min="0"` / `max="목표량(미설정 시 3000ml)"` — 10ml 단위 연속 조절
- CSS custom property `--fill-pct`를 inline style로 주입 → `::-webkit-slider-runnable-track` background 그라디언트 실시간 갱신
- `linear-gradient(to top, #1976D2 var(--fill-pct), #DDEEFF var(--fill-pct))` — 채워진 구간(파랑)·빈 구간(연한 파랑) 실시간 표현

**크로스 브라우저 대응**
| 엔진 | 사용한 pseudo-element |
|------|-----------------------|
| Chrome / Safari | `::-webkit-slider-thumb`, `::-webkit-slider-runnable-track` |
| Firefox | `::-moz-range-thumb`, `::-moz-range-track`, `::-moz-range-progress` |

**thumb 디자인**
- 흰 원형(28px) + 파란 테두리(`border: 3px solid #1976D2`) + 그림자
- `:active` 시 `scale(1.2)` + 그림자 강조 → 잡는 피드백

**레이아웃 변경**
- `bottle-and-slider` flex row: 물병 SVG(120px) + 슬라이더 컬럼(8px track, 264px 높이)
- 슬라이더 최상단 목표량 레이블 / 최하단 `0` 레이블
- 원형 진행 링 제거 → 하단 `현재량ml / 목표량ml N%` 텍스트 한 줄로 단순화
- 물 채움 transition을 `0.55s → 0.25s`로 단축해 드래그 응답성 개선

### 디자인 결정

- **writing-mode 선택 이유**: `transform: rotate(-90deg)` 방식은 컨테이너 크기 계산이 틀어지고 터치 이벤트 방향도 뒤집힘. `writing-mode: vertical-lr`은 브라우저가 레이아웃·이벤트를 모두 올바르게 처리.
- **track 배경 vs ::-webkit-slider-runnable-track**: `::-webkit-slider-runnable-track`에 `background: transparent`을 주고 요소 자체의 background에 그라디언트를 적용하면 CSS custom property 값이 pseudo-element까지 상속되어 실시간 업데이트가 가능.
